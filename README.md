# b0t

A personal local-AI tool for iOS, with companion styling. On-device, markdown-driven, single user, single device.

The user's b0t lives as a directory of plain markdown files in their Documents folder. A local language model animates those files — Apple Foundation Models by default where the device supports it, or a downloadable open-weight model the user chooses and owns. The app is a body for files the user owns. Productivity can be fun.

## Status

v1 is in active development. Phases 0–4 have shipped (project setup, markdown brain, the inference loop, module bridges + Tools, and the anatomical GUI with a static face); 279 tests pass. Phase 5 (onboarding) is deferred. The 2026-05-29 amendment re-opens Phase 2 to make inference engine-agnostic — Apple Foundation Models as the default where available, downloadable open-weight models (via llama.cpp) otherwise, switchable everywhere — and contracts v1 to a single non-modular b0t (modular face, multi-b0t, and unlock economy → v2). See `docs/IMPLEMENTATION.md` for the live cross-phase tracker and `docs/plans/amendment-2026-05-29-interpretation.md` for the amendment's impact map.
